## Common Issues

<AccordionGroup>
  <Accordion title="Database Connection Error">
    **Error:** `could not connect to server: Connection refused`
    
    **Solutions:**
    ```bash
    # Check PostgreSQL is running
    docker ps | grep postgres
    
    # Start PostgreSQL
    docker-compose up -d postgres
    
    # Check connection
    psql -h localhost -U vmledger -d vmledger
    ```
  </Accordion>
  
  <Accordion title="Redis Connection Error">
    **Error:** `Error connecting to Redis`
    
    **Solutions:**
    ```bash
    # Check Redis is running
    docker ps | grep redis
    
    # Start Redis
    docker-compose up -d redis
    
    # Test connection
    redis-cli ping
    ```
  </Accordion>
  
  <Accordion title="Migration Errors">
    **Error:** `Target database is not up to date`
    
    **Solutions:**
    ```bash
    # Run migrations
    alembic upgrade head
    
    # Reset database (WARNING: deletes all data)
    alembic downgrade base
    alembic upgrade head
    ```
  </Accordion>
  
  <Accordion title="Import Errors">
    **Error:** `ModuleNotFoundError: No module named 'vmledger'`
    
    **Solutions:**
    ```bash
    # Activate virtual environment
    source venv/bin/activate
    
    # Reinstall dependencies
    pip install -r requirements.txt
    
    # Add to PYTHONPATH
    export PYTHONPATH="${PYTHONPATH}:$(pwd)"
    ```
  </Accordion>
  
  <Accordion title="Frontend Build Errors">
    **Error:** `Module not found` or `Cannot find module`
    
    **Solutions:**
    ```bash
    cd frontend
    
    # Clear cache
    rm -rf .next node_modules
    
    # Reinstall
    npm install
    
    # Rebuild
    npm run build
    ```
  </Accordion>
  
  <Accordion title="Celery Not Starting">
    **Error:** `Celery worker not processing tasks`
    
    **Solutions:**
    ```bash
    # Check Redis connection
    redis-cli ping
    
    # Start worker with debug logging
    celery -A vmledger.celery_app worker --loglevel=debug
    
    # Check Beat scheduler
    celery -A vmledger.celery_app beat --loglevel=debug
    ```
  </Accordion>
  
  <Accordion title="DNS / Hostname Resolution Failing in Docker">
    **Error:** `Could not resolve hostname for IP` — works with `nslookup` on host but fails inside the app container.
    
    **Cause:** Docker containers default to Google's DNS (`8.8.8.8`) which can't resolve private/internal hostnames.
    
    **Solution:** Add your host's DNS servers to the `api` and `celery-worker` services in `docker-compose.yml`:
    ```yaml
    services:
      api:
        dns:
          - 10.208.210.1    # Your local DNS
          - 8.8.8.8         # Fallback
      celery-worker:
        dns:
          - 10.208.210.1
          - 8.8.8.8
    ```
    
    Then restart:
    ```bash
    docker-compose down && docker-compose up -d
    ```
  </Accordion>
  
  <Accordion title="Browser Autofill Turns Input Backgrounds White">
    **Symptom:** After selecting an autofill suggestion in Chrome/Edge, form input backgrounds become white, clashing with the dark theme.
    
    **Cause:** WebKit browsers apply their own background color to autofilled inputs.
    
    **Solution:** Add these CSS overrides to `globals.css`:
    ```css
    .input-premium:-webkit-autofill,
    .input-premium:-webkit-autofill:hover, 
    .input-premium:-webkit-autofill:focus, 
    .input-premium:-webkit-autofill:active {
      -webkit-box-shadow: 0 0 0 30px #111827 inset !important;
      -webkit-text-fill-color: #f3f4f6 !important;
      transition: background-color 5000s ease-in-out 0s;
    }
    ```
    
    This is already applied in VMLedger's `globals.css`.
  </Accordion>
</AccordionGroup>

## Debug Mode

### Enable Debug Logging

```python
# In .env
DEBUG=true
LOG_LEVEL=DEBUG
```

### View Logs

```bash
# API logs
tail -f logs/vmledger.log

# Docker logs
docker-compose logs -f api

# Celery logs
docker-compose logs -f celery-worker
```
