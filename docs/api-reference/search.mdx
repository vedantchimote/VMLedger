## Overview

The Search API provides powerful full-text search with **prefix matching** across IP addresses, hostnames, domains, tags, and deployment notes. Uses PostgreSQL's `to_tsquery('simple', ...)` with `:*` suffix for prefix matching, plus an ILIKE fallback for substring matches.

## Search VMs

<api method="GET" endpoint="/api/vms/search" />

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| q | string | Yes | Search query (min 1 character) |
| limit | integer | No | Max results (default: 50, max: 100) |

### Response (200 OK)

```json
{
  "success": true,
  "data": {
    "results": [
      {
        "id": 123,
        "hostname": "web-server-01",
        "ip_address": "192.168.1.100",
        "tags": ["production", "nginx"],
        "rank": 0.95,
        "highlighted_notes": "...Installed Software\n- <mark>Nginx</mark> 1.24.0..."
      }
    ],
    "count": 1,
    "query": "nginx"
  }
}
```

### Search Features

```mermaid
flowchart TD
    A[Search Query: "ngin"] --> B{Full-Text Search}
    B -->|Prefix match| C[tsquery 'ngin:*']
    
    A --> D{ILIKE Fallback}
    D -->|Substring match| E['%ngin%']
    
    C --> F((PostgreSQL DB))
    E --> F
    
    F --> G[Rank Results]
    G -->|Highest score| H[1. Nginx proxy]
    G -->|Partial score| I[2. app-staging]
```

- **Prefix matching**: `harbor` matches `harbornode`, `harbor-vm`, etc. (`:*` suffix in tsquery)
- **ILIKE fallback**: Catches mid-word substrings that tsquery misses (e.g. `node` in `workernode4`)
- **Multi-term search**: `nginx production` uses OR logic — matches either term
- **Relevance ranking**: FTS results ranked first, ILIKE-only matches appended at lower rank
- **Highlighting**: Matches in deployment notes wrapped in `<mark>` tags
- **Simple config**: Uses `'simple'` text search config (no stemming) for reliable partial matching
- **Fast performance**: &lt; 500ms for 1000 VMs

### Code Examples

<CodeGroup>

```bash cURL
curl "http://localhost:8000/api/vms/search?q=nginx" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```python Python
response = requests.get(
    "http://localhost:8000/api/vms/search",
    params={"q": "nginx production"},
    headers={"Authorization": f"Bearer {token}"}
)

results = response.json()['data']
for vm in results:
    print(f"{vm['hostname']} - Rank: {vm['rank']:.2f}")
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:8000/api/vms/search?q=nginx',
  { headers: { 'Authorization': `Bearer ${token}` } }
);

const results = await response.json().data;
results.forEach(vm => {
  console.log(`${vm.hostname} - Rank: ${vm.rank.toFixed(2)}`);
});
```
