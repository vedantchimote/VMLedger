## Overview

VMLedger uses Redis for caching, session storage, and as a message broker for Celery.

## Cache Keys

```
# Session tokens
session:{token_id}

# Dashboard cache
dashboard:{user_id}

# Rate limiting
ratelimit:{user_id}:{endpoint}

# User lockouts
lockout:{username}
```

## TTL Configuration

| Cache Type | TTL | Purpose |
|------------|-----|---------|
| Session tokens | 24 hours | Authentication |
| Dashboard data | 30 seconds | Performance |
| Rate limit counters | 1 minute | API protection |
| User lockouts | 30 minutes | Security |
